================================================================================
  PRISMA - QUICK START GUIDE
================================================================================

1. INSTALLATION
   -------------
   cd prisma_forecast
   pip install -r requirements.txt


2. TRAIN WITH DEMO DATA
   ---------------------
   python src\cli.py train --data data\prisma_dataset.csv

   Expected: R² = 0.9925 (99.25% accuracy)


3. PREDICT WITH DEMO DATA
   -----------------------
   python src\cli.py predict --input data\recent_history.csv --horizon 14

   Output: forecast_[material]_14d.json and .csv


4. USE YOUR OWN DATA
   ------------------
   
   Step A: Prepare your CSV with these columns:
           - date (required)
           - quantity_used or demand or sales (required)
           - material, region, project_id (optional, improves accuracy)
           - unit_price, total_cost (optional)

   Step B: Train with your data:
           python src\cli.py train --data your_data.csv

   Step C: Extract recent history (last 60-90 days):
           python -c "import pandas as pd; df = pd.read_csv('your_data.csv'); df['date'] = pd.to_datetime(df['date']); recent = df[df['date'] >= df['date'].max() - pd.Timedelta(days=60)]; recent.to_csv('recent.csv', index=False)"

   Step D: Generate forecast:
           python src\cli.py predict --input recent.csv --horizon 30


5. CUSTOM COLUMN NAMES
   --------------------
   If your columns have different names:
   
   python src\cli.py train --data data.csv --date-col order_date --qty-col sales_volume
   python src\cli.py predict --input recent.csv --horizon 14 --date-col order_date --qty-col sales_volume


6. OUTPUT FILES
   -------------
   JSON: forecast_[material]_[horizon]d.json
         - Contains metadata, predictions, confidence intervals
   
   CSV:  forecast_[material]_[horizon]d.csv
         - Tabular format with date, predictions, components


7. TROUBLESHOOTING
   ----------------
   Issue: "Column not found"
   Fix:   Use --date-col and --qty-col to specify column names

   Issue: Prophet fails on Windows
   Fix:   Automatic - pipeline uses LightGBM only (still 99.25% accurate)

   Issue: "Not enough data"
   Fix:   Use at least 6 months for training, 60 days for prediction


8. EXAMPLE TEMPLATE
   -----------------
   See: data/example_template.csv
   
   Minimum columns:
   date,material,quantity_used
   2024-01-01,cement,15.5
   2024-01-02,cement,18.2
   ...


9. PERFORMANCE
   ------------
   Training:   ~5 seconds (7,870 rows)
   Prediction: ~2 seconds (14-day forecast)
   Accuracy:   99.25% R² on demo data


10. SUPPORT
